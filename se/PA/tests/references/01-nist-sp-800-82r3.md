# NIST SP 800-82 Rev. 3

## Citation

National Institute of Standards and Technology (NIST), **SP 800-82 Rev. 3 — Guide to Operational Technology (OT) Security**, 2023.

## Why it is relevant here

The modeled field node is a remote OT asset with:

- direct sensing of physical-process variables,
- constrained communications over cellular WAN,
- strong dependence on availability and recoverability,
- hardware choices that directly affect cyber and operational risk.

NIST SP 800-82 is relevant because it frames OT design around the interaction between:

- control topology,
- communications architecture,
- reliability / redundancy assumptions,
- operational constraints,
- and security controls.

That is directly aligned with the user concern that OT design factors drive both system architecture and security needs.

## How this source is used in the test plan

This source supports scenario selection for:

- nominal remote telemetry upload,
- communications-loss and recovery behavior,
- secure transport and device identity checks,
- detection of degraded operation instead of silent failure,
- evidence that the node can operate for long periods with predictable behavior.

## Practical takeaways applied to `Field Node PA.d2`

- Because the architecture has no modeled redundancy, test focus shifts from failover to **fault detection, graceful degradation, and recovery**.
- Because the field node depends on cellular WAN, communications fault scenarios are first-class tests, not edge cases.
- Because the node is physically remote, long unattended operation and robust startup / restart behavior are critical acceptance topics.
- Because OT systems are sensitive to availability impacts, security tests for the upload path must avoid assumptions that would break operations in the field.

## Notes

Use this source as the high-level OT context reference, not as the sole basis for environmental or proven-in-use claims. Those require the IEC sources in this folder.
