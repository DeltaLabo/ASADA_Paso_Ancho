# ASADA_Paso_Ancho

## Toolset
* Code, Documentation, Task Management: GitHub
* CAD, CAM, PCB: Fusion 360
* Systems Engineering: Capella

## Project Life Cycle

**Current Stage:** Mission Definition Review

## Viewpoints

- Security
- Privacy
- Availability
- Mobile Accessibility
- Maintainability

## Requirements, Needs, and Desires

| Code | Requirement, Need, or Desire |
| --- | --- |
| RND1 | The aqueduct employees must be able to monitor water tanks through a dashboard accessible from any mobile device |
| RND2 | The aqueduct employees must be alerted of high overflow or undersupply risk |
| RND3 | The aqueduct employees must be assured that their data cannot be externally modified, deleted, or accessed by unauthorized parties |
| RND4 | The aqueduct employees may want to be assured that the system is physically secure against attackers |
| RND5 | The aqueduct employees may want to be assured that the system is cybernetically secure against attackers |
| RND6 | The aqueduct employees may want to be capable of conducting the full 38924-S standard water quality test suite (turbidity, free chlorine, E. coli, pH, transition metals, etc.) |
| RND7 | The aqueduct employees should be assured that the risk of dangerous systematic faults is negligibly low |
| P1 | The system must obtain its energy from a solar panel. |
| P2 | The system must have a battery with an autonomy of at least 3 months. |
| P3 | The system must be replicable using the bill of materials (BOM) |
| M1 | The system must be contained in a cabinet that can withstand environmental conditions |
| M2 | The system should have a lock mechanism |
| M3 | The system should operate for at least 48 continuous hours under direct exposure to rain or sunlight |
| C1 | The system must implement mobile connectivity infrastructure |
| C2 | The system must avoid mobile connectibity interruptions |
| CS1 | The system must be cybersecurity-hardened following applicable standards |
